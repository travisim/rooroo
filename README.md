# Roostoo Trading Bot

Autonomous cryptocurrency trading bot for the Roostoo hackathon. Trades BTC/USD, ETH/USD, and SOL/USD on spot markets using systematic momentum and trend-following strategies backed by a full parameter sweep on 1 year of Binance OHLCV data.

## Strategy

The active strategy is configured in `config.py`. The default is **RSI mean-reversion (14) on 15m** with a 5% hard stop loss. After the backtest sweep, the bot is updated to the highest-composite-scoring configuration.

| Parameter | Value |
|-----------|-------|
| Strategy | RSI mean-reversion |
| Timeframe | 15m |
| RSI period | 14 |
| Entry | RSI crosses below 35 (oversold) |
| Exit | RSI crosses above 65 (overbought) |
| Stop loss | 5% below entry |
| Assets | BTC/USD, ETH/USD, SOL/USD (independent) |
| Position size | ~$280,000 per asset (≤ 84% portfolio) |
| Fees | 0.1% taker (MARKET orders) |

Available strategies: `ema_crossover`, `rsi`, `atr_trend`, `rsi_ema_filter`.

## Project Structure

```
├── config.py              # Strategy parameters (single source of truth)
├── roostoo_client.py      # Roostoo API client (HMAC auth, orders, balance)
├── bot/
│   ├── runner.py          # Main bot loop — autonomous, runs every 15 min
│   ├── indicators.py      # Pure-pandas indicators: EMA, RSI, ATR, crossovers
│   └── market_data.py     # Binance public REST API for live OHLCV
├── backtest/
│   ├── fetch_data.py      # Downloads 1-year OHLCV from Binance (7 timeframes)
│   ├── sweep.py           # Parameter sweep ("ralph loop") — ~1,500 configurations
│   └── walk_forward.py    # Out-of-sample validation of best sweep result
├── data/                  # Cached Binance OHLCV (HDF5, git-ignored)
└── results/               # sweep_results.csv, best_params.json, walk_forward.json
```

## Setup

### Live Bot (EC2 deployment)

```bash
cp .env.example .env
# Fill in ROOSTOO_API_KEY and ROOSTOO_API_SECRET

pip install -r requirements.txt

# Run locally:
python bot/runner.py

# Deploy to EC2:
./deploy.sh
```

### Backtest & Strategy Optimisation (requires vbtpro conda env)

```bash
# Install extra dependency in vbtpro env
conda run -n vbtpro pip install -r requirements_vbt.txt

# Step 1: Download 1 year of OHLCV data for all timeframes (~20 min)
conda run -n vbtpro python backtest/fetch_data.py

# Step 2: Run parameter sweep — stops early when composite > 1.5
conda run -n vbtpro python backtest/sweep.py --timeframes 1h 4h

# Full sweep across all timeframes (~2 hours):
conda run -n vbtpro python backtest/sweep.py --all-timeframes

# Step 3: Validate best config on held-out data
conda run -n vbtpro python backtest/walk_forward.py

# Step 4: Update config.py → redeploy
./deploy.sh
```

## Optimisation Metric

The sweep optimises for the **competition composite score**:

```
Composite = 0.4 × Sortino + 0.3 × Sharpe + 0.3 × Calmar
```

Subject to: `trades/day ≥ 1`, `max_drawdown < 25%`. The sweep stops early (ralph loop) when a configuration exceeds the composite threshold of 1.5.

## Risk Management

- **Stop loss**: Hard stop at 5% below entry price (configurable via `STOP_LOSS_PCT`)
- **Take profit**: Optional, disabled by default — strategy exit signal used instead
- **Max exposure**: 3 × $280K = $840K of $1M portfolio (84%)
- **No pyramiding**: One position per asset at a time
- **Spot only**: No leverage, no short selling
- **Fee awareness**: Strategies require >0.2% edge per round trip to beat 2× taker fee
